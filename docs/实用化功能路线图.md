# GIS AI 算法库实用化功能路线图

## 一、文档目的

本文档用于回答三个核心问题：

1. 当前项目处于什么阶段
2. 从实用化角度，项目未来应重点补强哪些能力
3. 应该沿着什么产品与技术路线演进

## 二、当前项目定位（2026-05-15 更新）

项目已完成产品化核心阶段，从"算法库雏形"升级为**可工作的遥感智能解译引擎**。

已覆盖的能力：
- Core 基础能力 + 结构化错误码
- Raster / Vector / PointCloud IO（含 GeoPackage）
- 常见 GIS 算法
- ONNX Runtime 推理封装
- 遥感大图语义分割全流程（滑窗推理 + 融合 + 后处理）
- 批量处理
- C API + CLI + Qt GUI
- JSON 任务配置 + 运行报告

当前定位：**面向遥感智能解译方向的 GIS AI 基础引擎**

## 五、路线图结论

核心判断仍然有效：**不应优先追求"更多零散功能"，而应优先补齐"遥感解译完整流程能力 + 可交付工程能力"。**

前三个阶段（工程闭环 → 场景打透 → 产品化）已基本完成。下一步应按需扩展功能模块，同时保持核心流程的稳定性和可交付性。
