# GIS AI TOOL 架构设计文档

## 1. 概述

GIS AI TOOL 是一个基于 C++17 构建的遥感智能解译工具，集成深度学习推理与传统 GIS 算法，提供 CLI（命令行界面）和 GUI（图形用户界面）双入口，底层共享同一套算法核心库。

### 技术栈

| 组件 | 技术 | 版本 |
|------|------|------|
| 语言 | C++17 | — |
| 构建系统 | CMake | ≥3.21 |
| 包管理器 | vcpkg | — |
| GIS 核心 | GDAL / GEOS / PROJ | 3.6+ / 3.11+ / 9.2+ |
| AI 推理 | ONNX Runtime | 1.16+ |
| GUI 框架 | Qt6 | 6.x（Widgets + Svg + Sql） |
| 日志 | spdlog | 1.12+ |
| 测试框架 | Google Test | 1.14+ |

### 设计原则

1. **分层解耦**：核心算法库、IO、AI 推理、融合层各自独立，通过清晰接口交互
2. **双入口共享**：CLI 和 GUI 共享同一套核心算法库，参数体系一致
3. **融合驱动**：Fusion 层编排 AI 推理与 GIS 处理，实现端到端流程
4. **进度可观测**：所有耗时操作支持进度回调
5. **线程安全**：批量处理支持多线程并行，推理引擎独立实例

### 2.1 核心层 (core)

基础能力模块，为上层提供通用支撑。

| 模块 | 文件 | 职责 |
|------|------|------|
| 日志 | `logger.h/cpp` | 基于 spdlog 的全局日志系统，支持文件输出和级别控制 |
| 异常 | `exception.h/cpp` | 统一异常体系（GisAiAlgorithmException / GisAiModelException） |
| 配置 | `config.h/cpp` | 全局配置管理 |
| 平台 | `platform.h/cpp` | 跨平台兼容 |
| 内存 | `memory.h/cpp` | 内存管理辅助 |
| 导出 | `export.h` | DLL 导出宏定义 |

### 2.2 IO 层 (io)

数据读写模块，封装 GDAL/OGR API。

| 模块 | 文件 | 职责 |
|------|------|------|
| 栅格 IO | `raster_io.h/cpp` | 栅格数据加载与保存（GeoTIFF 等） |
| 矢量 IO | `vector_io.h/cpp` | 矢量数据加载与保存（Shapefile、GeoJSON） |
| 点云 IO | `pointcloud_io.h/cpp` | 基于 OGR 的点要素读写 |
| IO 工厂 | `io_factory.h/cpp` | IO 实例统一创建 |

### 2.3 GIS 层 (gis)

传统 GIS 算法模块。

| 模块 | 文件 | 职责 |
|------|------|------|
| 栅格重采样 | `raster_resample.h/cpp` | Nearest / Bilinear 重采样 |
| 栅格归一化 | `raster_normalize.h/cpp` | Min-Max 归一化（自动 / 指定范围） |
| 栅格裁剪 | `raster_clip.h/cpp` | 地理范围裁剪 / 像素偏移裁剪 |
| 栅格镶嵌 | `raster_mosaic.h/cpp` | 5 种镶嵌策略（Overwrite/First/Mean/Max/Min） |
| 栅格阈值分割 | `raster_threshold.h/cpp` | 单阈值 / 范围阈值分割 |
| 矢量简化 | `vector_simplify.h/cpp` | 基于 GEOS 的 Douglas-Peucker 简化 |
| 矢量缓冲区 | `vector_buffer.h/cpp` | 基于 GEOS 的缓冲区生成 |
| 矢量裁剪 | `vector_clip.h/cpp` | 基于 GEOS 的矢量裁剪 |
| 矢量相交 | `vector_intersect.h/cpp` | 矢量相交运算 |
| 矢量拓扑 | `vector_topology.h/cpp` | 拓扑检查与修复 |
| 坐标转换 | `coord_transform.h/cpp` | 基于 PROJ 的 CRS 转换 |
| GEOS 工具 | `geos_utils.h` | GEOS 上下文管理与几何转换辅助 |

### 2.4 AI 层 (ai)

深度学习推理模块。

| 模块 | 文件 | 职责 |
|------|------|------|
| ONNX 上下文 | `ort_context.h/cpp` | ONNX Runtime 环境初始化与会话管理 |
| 模型管理 | `model_manager.h/cpp` | 模型加载、缓存与元数据查询 |
| 推理引擎 | `inference_engine.h/cpp` | 模型推理执行（单输入 / 多输入） |
| 预处理 | `preprocess.h/cpp` | 栅格转张量、5 种归一化模式 |
| 后处理 | `postprocess.h/cpp` | Sigmoid + Argmax、阈值掩膜、掩膜转栅格 |
| 掩膜转矢量 | `mask_to_polygon.h/cpp` | 8 方向 Moore 邻域追踪 + 内环检测 |

### 2.5 融合层 (fusion)

端到端流程编排模块，组合 AI 推理与 GIS 处理。

| 模块 | 文件 | 职责 |
|------|------|------|
| 大图分割 | `large_image_seg.h/cpp` | 滑窗推理 + 重叠区混合 + 矢量化 |
| 栅格分割 | `raster_seg.h/cpp` | 单图分割流程 |
| 批量处理 | `batch_processor.h/cpp` | 多线程批量推理 |
| 任务配置 | `task_config.h/cpp` | 任务配置加载与执行报告 |

## 3. 数据流

### 3.1 大图分割数据流

```
输入栅格 → 分块计算 → 逐块提取
    → 预处理（归一化 + Resize）
    → ONNX 推理
    → 后处理（Sigmoid + Argmax）
    → 重叠区混合（None/Linear/Gaussian）
→ 合成完整掩膜
→ 掩膜转矢量（8 方向追踪 + 内环检测）
→ 过滤小多边形 → 简化 → 拓扑修复 → 属性添加
→ 输出栅格 + 矢量
```

### 3.2 批量处理数据流

```
输入目录/文件列表
    → 文件扫描（*.tif / *.tiff）
    → 工作线程池（每个线程独立 RasterSeg 实例）
        → 单图分割流程
    → 汇总结果
→ BatchResult 列表
```

### 3.3 GUI 参数流

```
用户交互 → ParamWidget 参数收集 → TaskConfig
    → TaskRunner::Execute()
    → LargeImageSeg / BatchProcessor
    → 进度回调 → QtProgressReporter → UI 更新
→ TaskReport → 结果展示
```

## 5. 线程模型

### GUI 线程

- **主线程**：Qt 事件循环，负责 UI 渲染和交互
- **工作线程**：`ExecuteWorker` 在 `QThread` 中执行算法，通过信号槽更新 UI
- **进度回调**：`QtProgressReporter` 通过 Qt 信号槽机制跨线程更新 UI

### 批量处理线程

- **主线程**：任务调度与进度汇总
- **工作线程池**：`BatchProcessor` 创建 `num_threads` 个工作线程
- **独立推理实例**：每个工作线程拥有独立的 `RasterSeg` 实例，避免 ONNX Runtime 会话冲突
- **原子索引**：`std::atomic<size_t>` 实现无锁任务分发
- **互斥回调**：`std::mutex` 保护进度回调的线程安全

### CLI 线程

- 单线程执行，通过命令行参数触发后直接调用核心 API

## 6. 构建与安装

### 6.1 构建

```bash
# Windows (vcpkg 全局安装模式)
cmake -B build -S . -DCMAKE_TOOLCHAIN_FILE=%VCPKG_ROOT%/scripts/buildsystems/vcpkg.cmake
cmake --build build --config Release

# Linux
cmake -B build -S . -DCMAKE_TOOLCHAIN_FILE=$VCPKG_ROOT/scripts/buildsystems/vcpkg.cmake
cmake --build build
```

### 6.2 CMake 选项

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `GIS_AI_BUILD_GUI` | 构建 Qt6 GUI | OFF |
| `GIS_AI_BUILD_TESTS` | 构建测试 | ON |
| `GIS_AI_BUILD_EXAMPLES` | 构建示例 | ON |
| `GIS_AI_BUILD_TOOLS` | 构建工具和 CLI | ON |
| `GIS_AI_BUILD_SHARED` | 构建动态库 | ON |
| `GIS_AI_BUILD_STATIC` | 构建静态库 | ON |
| `GIS_AI_ENABLE_CUDA` | 启用 CUDA 推理 | OFF |
| `GIS_AI_BUILD_DOCS` | 构建 Doxygen 文档 | OFF |

### 6.3 安装

```bash
cmake --install build --prefix install
```
