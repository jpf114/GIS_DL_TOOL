# GIS AI TOOL 用户手册

## 1. 简介

GIS AI TOOL（遥感智能解译工具）是一个基于深度学习的遥感影像处理工具，集成 AI 推理与传统 GIS 算法，提供 **6 大功能**，覆盖从数据预处理、模型推理到后处理输出的完整链路。支持命令行（CLI）和图形界面（GUI）两种使用方式。

## 2. 安装与启动

### 2.1 环境要求

- **操作系统**：Windows 10+ / Linux
- **运行时**：Visual C++ 运行时库（Windows）
- **模型文件**：ONNX 格式的语义分割模型

### 2.2 安装

将 `install/` 目录复制到目标位置，目录结构如下：

```
install/
├── bin/
│   ├── gis_ai_cli.exe       # 命令行入口
│   ├── gis-ai-gui.exe       # 图形界面入口
│   ├── gis_ai.dll           # 核心算法库
│   ├── onnxruntime.dll      # ONNX Runtime
│   ├── gdal.dll             # GDAL
│   ├── geos_c.dll           # GEOS
│   ├── proj_9.dll           # PROJ
│   └── ...                  # 其他运行时依赖
└── share/
    ├── icons/               # SVG 图标资源
    ├── proj/                # PROJ 数据
    └── gdal/                # GDAL 数据
```

### 2.3 验证安装

```bash
# 命令行验证
gis_ai_cli version

# 查看帮助
gis_ai_cli help
```

1. **选择功能**：点击左侧导航栏中的功能项

- **操作按钮**：重新执行、编辑参数、删除

## 4. 命令行 (CLI) 使用指南

### 4.1 基本命令

```bash
# 查看帮助
gis_ai_cli help

# 查看版本
gis_ai_cli version
```

### 4.2 单图分割

```bash
gis_ai_cli segment <model_path> <input_path> [output_tif] [output_shp] [options]
```

示例：

```bash
# 基本用法
gis_ai_cli segment model.onnx input.tif output_seg.tif output_seg.shp

# 自定义切片参数
gis_ai_cli segment model.onnx input.tif output_seg.tif output_seg.shp --tile-size=256 --stride=192

# 使用线性混合
gis_ai_cli segment model.onnx input.tif output_seg.tif output_seg.shp --blend=linear

# 只输出栅格，不输出矢量
gis_ai_cli segment model.onnx input.tif output_seg.tif --no-shp
```

### 4.3 批量分割

```bash
gis_ai_cli batch <model_path> <input_dir> <output_dir> [options]
```

示例：

```bash
# 4 线程批量处理
gis_ai_cli batch model.onnx input_dir/ output_dir/ --threads=4

# 自定义参数
gis_ai_cli batch model.onnx input_dir/ output_dir/ --threads=2 --tile-size=256 --stride=192
```

### 4.4 使用配置文件

```bash
# 生成默认配置模板
gis_ai_cli generate-config task_config.json

# 编辑配置文件后执行
gis_ai_cli run task_config.json
```

### 4.5 CLI 选项说明

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `--tile-size <int>` | 切片大小（像素） | 512 |
| `--stride <int>` | 滑窗步长（像素） | 384 |
| `--blend <mode>` | 融合模式：none / linear / gaussian | gaussian |
| `--threads <int>` | 批量处理线程数 | 1 |
| `--no-shp` | 不输出矢量文件 | — |
| `--verbose` | 详细日志输出 | — |

## 5. 功能说明

### 5.1 大图分割

对超出模型输入尺寸的大幅遥感影像进行滑窗推理，支持重叠区混合和矢量化输出。

### 5.2 模型推理

基于 ONNX Runtime 执行语义分割模型推理，支持多类别输出。

### 5.3 数据预处理

| 功能 | 说明 |
|------|------|
| 栅格重采样 | 调整栅格分辨率（Nearest / Bilinear） |
| 栅格归一化 | 将像元值归一化到 [0,1] 区间 |
| 栅格裁剪 | 按地理范围或像素范围裁剪栅格 |

### 5.4 矢量处理

| 功能 | 说明 |
|------|------|
| 矢量简化 | Douglas-Peucker 算法减少顶点 |
| 矢量缓冲区 | 按指定距离生成缓冲区 |
| 矢量裁剪 | 用裁切图层裁剪目标图层 |

### 5.5 栅格处理

| 功能 | 说明 |
|------|------|
| 栅格镶嵌 | 多幅栅格拼接（5 种策略） |
| 栅格阈值分割 | 按阈值二值化 |
| 掩膜转矢量 | 将二值掩膜转为矢量多边形 |

### 5.6 批量处理

对目录下多个 TIFF 文件执行批量推理，支持多线程并行。

## 7. 常见问题

### 7.1 启动报错"Model file not found"

**原因**：模型文件路径不正确。

**解决**：检查模型文件路径是否正确，确保使用 ONNX 格式（`.onnx`）的模型文件。

### 7.2 推理结果全为同一类别

**可能原因**：
- 模型输入尺寸与实际配置不匹配
- 归一化参数不正确（如 ImageNet 均值/标准差与训练时不一致）
- 输入数据类型配置错误（`input_is_uint8` 设置不当）

**解决**：确认模型的输入尺寸、归一化参数与训练时一致。

### 7.3 大图分割接缝明显

**原因**：重叠区混合模式设置不当。

**解决**：
- 使用 `Gaussian` 混合模式（默认），效果最佳
- 增大步长与切片大小的差值（即增大重叠区域），如 `tile_size=512, stride=256`

### 7.4 批量处理速度慢

**解决**：
- 增加线程数：`--threads=4`（建议不超过 CPU 核心数）
- 使用 Release 构建：Debug 模式下 ONNX Runtime 性能显著低于 Release

### 7.5 内存不足

**可能原因**：输入影像过大或切片尺寸过大。

**解决**：
- 减小 `tile_size`（如 256）
- 使用 `skip_nodata` 跳过全 NoData 区域
- 分批处理

### 7.6 ONNX Runtime 相关错误

**原因**：ONNX Runtime 在 Windows Debug 模式下存在兼容性问题。

**解决**：使用 Release 模式构建和运行。
