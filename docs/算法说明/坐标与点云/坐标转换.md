# 坐标与点云 / 坐标转换

## 功能说明

对坐标、矢量数据和点云数据进行坐标系转换。支持 EPSG、WKT 等 CRS 格式，基于 PROJ 库实现。

## 输入与输出

- 输入：坐标点 / 矢量数据 / 点云数据，源坐标系和目标坐标系
- 输出：转换后的坐标点 / 矢量数据 / 点云数据

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| x | double | 是 | 无 | X 坐标（单点转换） |
| y | double | 是 | 无 | Y 坐标（单点转换） |
| z | double | 否 | 0.0 | Z 坐标（单点转换） |
| src_crs | string | 是 | 无 | 源坐标系，支持 EPSG（如 "EPSG:4326"）、WKT 格式 |
| dst_crs | string | 是 | 无 | 目标坐标系，支持 EPSG（如 "EPSG:3857"）、WKT 格式 |
| coords | vector\<CoordPair\> | 是 | 无 | 坐标点集合（批量转换） |
| VectorData | VectorData | 是 | 无 | 矢量数据（矢量转换） |
| PointCloudData | PointCloudData | 是 | 无 | 点云数据（点云转换） |

## 算法内容

使用 PROJ 库实现坐标转换，提供 C API 接口 `GisAi_TransformCoordinates`。

### 类：CoordTransform

#### Transform(x, y, z, src_crs, dst_crs) -> CoordPair

单点坐标转换。根据源坐标系和目标坐标系创建 PROJ 转换对象，对单个坐标点执行转换。

#### TransformBatch(coords, src_crs, dst_crs) -> vector\<CoordPair\>

批量坐标转换。对一组坐标点执行相同坐标系转换，内部复用 PROJ 转换对象，提高批量转换效率。

#### TransformVector(VectorData, dst_crs) -> unique_ptr\<VectorData\>

矢量数据坐标转换。遍历矢量数据中所有要素的几何坐标（含内环 inner_rings），逐一执行转换，属性字段不变。

#### TransformPointCloud(PointCloudData, dst_crs) -> unique_ptr\<PointCloudData\>

点云数据坐标转换。遍历点云中所有点的 XYZ 坐标，执行批量转换。

### 处理细节

- 使用 PROJ 的 `proj_create_crs_to_crs` 创建转换管道。
- 使用 `proj_trans` 执行坐标转换。
- 支持 EPSG 代码（如 "EPSG:4326"）和 WKT 字符串两种 CRS 格式。
- 矢量转换时同步处理外环和内环（inner_rings）坐标。
- 点云转换时保留强度、分类等非坐标属性。

### C API

```c
int GisAi_TransformCoordinates(
    const char* src_crs,
    const char* dst_crs,
    double* x, double* y, double* z,
    int count
);
```

## 处理流程

1. 解析源坐标系和目标坐标系字符串。
2. 创建 PROJ 转换对象（`proj_create_crs_to_crs`）。
3. 根据数据类型执行转换：
   - **单点**：调用 `proj_trans` 转换单个坐标。
   - **批量**：循环调用 `proj_trans` 转换所有坐标点。
   - **矢量**：遍历要素几何的所有环（含内环），逐点转换。
   - **点云**：遍历所有点，批量转换 XYZ 坐标。
4. 销毁 PROJ 转换对象。
5. 返回转换结果。

## 输出说明

- 单点转换返回 CoordPair（x, y, z）
- 批量转换返回坐标点向量
- 矢量转换返回新矢量数据，几何坐标已转换，属性不变
- 点云转换返回新点云数据，XYZ 已转换，其他属性不变

## 结果解释

- 输出坐标单位由目标坐标系决定（如 EPSG:3857 为米，EPSG:4326 为度）
- 转换精度取决于 PROJ 库的转换模型和参数
- 大地高与正常高之间的转换需要额外高程异常数据

## 适用场景

- 将 WGS84 经纬度数据转换为投影坐标系
- 不同投影之间的数据拼接与叠加
- 点云数据与矢量数据统一坐标系
- GIS 数据入库前的坐标标准化

## 推荐输出文件名

- 矢量：`xxx_reprojected.gpkg`
- 点云：`xxx_reprojected.las`

## 注意事项

- 源坐标系必须与数据实际坐标系一致，否则转换结果错误
- 跨带投影可能导致变形较大，需选择合适的投影带
- 某些坐标系转换需要 PROJ 数据网格文件（如 NTv2 格网），缺失时转换可能失败
- 矢量转换时内环（inner_rings）坐标也会被转换，无需额外处理
- 批量转换效率远高于逐点调用单点转换
