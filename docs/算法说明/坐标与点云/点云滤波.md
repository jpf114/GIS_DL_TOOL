# 坐标与点云 / 点云滤波

## 功能说明

对点云数据进行滤波处理，移除噪声点或不在指定范围内的点。支持直通滤波和统计离群值移除两种方式。

## 输入与输出

- 输入：点云数据
- 输出：滤波后的点云数据

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入点云 | PointCloudData | 是 | 无 | 输入点云数据 |
| axis | string | 否 | "z" | 直通滤波的过滤轴（x / y / z） |
| min_val | double | 否 | 0.0 | 直通滤波的最小值 |
| max_val | double | 否 | 100.0 | 直通滤波的最大值 |
| mean_k | int | 是 | 无 | SOR 的近邻点数 |
| stddev_mul_thresh | double | 是 | 无 | SOR 的标准差倍数阈值 |

### PassFilterParams 结构

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| axis | string | "z" | 过滤轴（x / y / z） |
| min_val | double | 0.0 | 范围最小值 |
| max_val | double | 100.0 | 范围最大值 |

## 算法内容

### 类：PcFilter

#### PassThrough(PointCloudData, PassFilterParams) -> unique_ptr\<PointCloudData\>

直通滤波。按指定轴的坐标范围过滤点云，仅保留该轴坐标在 [min_val, max_val] 范围内的点。

#### StatisticalOutlierRemoval(PointCloudData, mean_k, stddev_mul_thresh) -> unique_ptr\<PointCloudData\>

统计离群值移除（SOR）。基于每个点的 k 近邻平均距离的统计分布，移除离群点。

### 直通滤波原理

1. 遍历点云中所有点。
2. 对每个点，读取指定轴（x / y / z）的坐标值。
3. 若坐标值在 [min_val, max_val] 范围内，保留该点；否则移除。
4. 输出保留的点构成的点云。

### SOR 原理

1. 对点云中每个点，搜索其 mean_k 个最近邻点。
2. 计算该点到 k 个近邻点的平均距离。
3. 计算所有点的平均距离的均值 μ 和标准差 σ。
4. 若某点的平均距离 > μ + stddev_mul_thresh × σ，则判定为离群点并移除。
5. 输出保留的点构成的点云。

### 处理细节

- 直通滤波为简单范围过滤，计算速度快。
- SOR 需要构建空间索引（如 KD 树）进行近邻搜索，计算量较大。
- 两种方式均保留点的所有属性（强度、分类等）。
- SOR 的 mean_k 值越大，统计越稳定，但计算越慢。

## 处理流程

1. 验证输入点云非空。
2. 根据调用方法执行对应滤波：
   - **直通滤波**：
     a. 读取 PassFilterParams 参数。
     b. 遍历所有点，按指定轴范围过滤。
     c. 汇总保留的点。
   - **SOR**：
     a. 构建 KD 树空间索引。
     b. 对每个点搜索 mean_k 个近邻。
     c. 计算每个点的近邻平均距离。
     d. 计算全局均值和标准差。
     e. 移除超过阈值的离群点。
     f. 汇总保留的点。
3. 返回滤波结果。

## 输出说明

- 直通滤波输出点数为范围内点的数量
- SOR 输出点数为移除离群点后的数量
- 输出点云属性与输入一致

## 结果解释

- 直通滤波移除了指定轴范围外的点，适合按高程或平面范围裁剪
- SOR 移除了统计意义上的离群点，适合去除噪声
- stddev_mul_thresh 越小，移除的点越多，滤波越激进
- mean_k 越大，近邻统计越稳定，但计算越慢

## 适用场景

- 去除点云中的地面以下或高空噪声点（直通滤波）
- 按高程范围提取特定层位的点（直通滤波）
- 去除点云采集中的散点噪声（SOR）
- 点云数据预处理，提升后续算法质量

## 推荐输出文件名

- `xxx_passthrough.las`
- `xxx_sor_filtered.las`

## 注意事项

- 直通滤波的 min_val 必须小于 max_val
- 直通滤波的 axis 仅支持 x / y / z
- SOR 的 mean_k 应大于 0，通常取 20~50
- SOR 的 stddev_mul_thresh 通常取 1.0~3.0，值越小移除越多
- SOR 对均匀分布的噪声效果较好，对局部密集噪声效果有限
- 大规模点云的 SOR 计算耗时较长，建议先降采样再滤波
