# 数据预处理 / 栅格归一化

## 功能说明

将栅格像元值归一化到 [0, 1] 区间，支持自动 Min-Max 和指定范围两种模式。

## 输入与输出

- 输入：栅格数据（GeoTIFF）
- 输出：归一化后的栅格数据，建议 `.tif`

## 参数

### Execute（自动 Min-Max）

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入栅格 | RasterData | 是 | 无 | 输入栅格数据 |

### ExecuteMinMax（指定范围）

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入栅格 | RasterData | 是 | 无 | 输入栅格数据 |
| min_val | float | 是 | 无 | 指定最小值 |
| max_val | float | 是 | 无 | 指定最大值 |

## 算法内容

### 自动 Min-Max 归一化

逐波段计算实际最小值和最大值（忽略 NaN），然后按公式归一化：

```
output = (value - min) / (max - min)
```

当波段值域为零（max - min < 1e-10）时，输出全部为 0。

### 指定范围归一化

使用用户指定的 min_val 和 max_val 进行归一化，超出范围的值会被 clamp 到 [min_val, max_val] 后再归一化：

```
clamped = max(min_val, min(max_val, value))
output = (clamped - min_val) / (max_val - min_val)
```

### NaN 处理

输入中的 NaN 值在输出中保持为 NaN，不参与归一化计算。

## 处理流程

1. 验证输入栅格是否有波段数据。
2. 逐波段处理：
   - 自动模式：扫描计算 min/max
   - 指定模式：使用用户提供的 min_val/max_val
3. 逐像元应用归一化公式。
4. 返回归一化结果。

## 输出说明

- 输出栅格波段数、尺寸、空间参考与输入一致
- 像元值范围 [0, 1]，NaN 保持不变
- 值域为零的波段输出全 0

## 结果解释

- 值 0 对应输入的最小值（或 min_val）
- 值 1 对应输入的最大值（或 max_val）
- 中间值线性映射到 [0, 1]

## 适用场景

- 为深度学习模型准备输入数据
- 统一不同量纲的栅格数据
- 可视化前将数据映射到 [0, 1]

## 推荐输出文件名

- `xxx_normalized.tif`

### C++ 接口

```cpp
namespace gis_ai {

class GIS_AI_API RasterNormalize {
public:
    RasterNormalize() = default;

    // 自动 Min-Max 归一化，将像元值归一化到 [0, 1]
    std::unique_ptr<RasterData> Execute(const RasterData& input);

    // 指定范围归一化，超出范围的值会被 clamp
    std::unique_ptr<RasterData> ExecuteMinMax(const RasterData& input, float min_val, float max_val);
};

} // namespace gis_ai
```

### 使用示例

```cpp
#include "gis/raster_normalize.h"
#include "io/raster_io.h"

// 读取输入栅格
auto raster = gis_ai::ReadRaster("input.tif");

// 方式一：自动 Min-Max 归一化
gis_ai::RasterNormalize normalizer;
auto result = normalizer.Execute(*raster);

// 方式二：指定范围归一化（如 NDVI 归一化到 [-1, 1]）
auto result2 = normalizer.ExecuteMinMax(*raster, -1.0f, 1.0f);

// 保存结果
gis_ai::WriteRaster(*result, "output_normalized.tif");
```

## 注意事项

- 自动 Min-Max 模式对异常值敏感，一个极端值可能导致大部分像元集中在很小的区间
- 指定范围模式会 clamp 超出范围的值，可能丢失极值信息
- NaN 值不参与 min/max 计算，也不被归一化
- max_val 必须大于 min_val，否则报错
