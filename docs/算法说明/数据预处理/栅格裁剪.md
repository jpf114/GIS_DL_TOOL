# 数据预处理 / 栅格裁剪

## 功能说明

按地理范围或像素偏移裁剪栅格数据，提取指定区域的子集。

## 输入与输出

- 输入：栅格数据（GeoTIFF）
- 输出：裁剪后的栅格数据，建议 `.tif`

## 参数

### Execute（地理范围裁剪）

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入栅格 | RasterData | 是 | 无 | 输入栅格数据 |
| bbox.min_x | double | 是 | 无 | 裁剪范围最小 X 坐标 |
| bbox.max_x | double | 是 | 无 | 裁剪范围最大 X 坐标 |
| bbox.min_y | double | 是 | 无 | 裁剪范围最小 Y 坐标 |
| bbox.max_y | double | 是 | 无 | 裁剪范围最大 Y 坐标 |

### ExecuteByPixel（像素偏移裁剪）

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入栅格 | RasterData | 是 | 无 | 输入栅格数据 |
| x_off | int | 是 | 无 | 起始列偏移（0 开始） |
| y_off | int | 是 | 无 | 起始行偏移（0 开始） |
| width | int | 是 | 无 | 裁剪宽度（像素） |
| height | int | 是 | 无 | 裁剪高度（像素） |

## 算法内容

### 地理范围裁剪

1. 将地理坐标 (min_x, max_y) 和 (max_x, min_y) 通过 GeoTransform 逆变换转换为像素坐标。
2. 将像素坐标 clamp 到栅格有效范围内。
3. 计算裁剪区域的像素偏移和尺寸。
4. 调用像素偏移裁剪完成实际裁剪。

GeoTransform 逆变换公式：

```
inv_det = gt[1]*gt[5] - gt[2]*gt[4]
px = round((dx*gt[5] - dy*gt[2]) / inv_det)
py = round((dy*gt[1] - dx*gt[4]) / inv_det)
```

### 像素偏移裁剪

1. 验证偏移和尺寸参数在有效范围内。
2. 更新输出 GeoTransform 的左上角坐标。
3. 逐波段逐行复制像素数据。

## 处理流程

1. 验证输入栅格有效性。
2. 地理范围模式：将 bbox 转为像素坐标。
3. 检查裁剪区域与栅格是否有交集。
4. 逐波段复制裁剪区域数据。
5. 更新 GeoTransform。
6. 返回裁剪结果。

## 输出说明

- 输出栅格波段数与输入一致
- 输出空间参考与输入一致
- GeoTransform 的左上角坐标更新为裁剪区域的左上角

## 结果解释

- 输出栅格仅包含裁剪区域内的像元
- 地理范围裁剪时，bbox 超出栅格范围的部分会被自动截断
- 裁剪区域与栅格无交集时报错

## 适用场景

- 从大幅影像中提取研究区域
- 为模型推理准备指定尺寸的输入
- 去除影像边缘无效区域

## 推荐输出文件名

- `xxx_clip.tif`

## 注意事项

- 地理范围裁剪时，bbox 坐标系必须与栅格一致
- 像素偏移裁剪时，偏移和尺寸不能超出栅格边界
- 裁剪区域与栅格无交集时会报错
- GeoTransform 退化（行列式接近零）时会报错
