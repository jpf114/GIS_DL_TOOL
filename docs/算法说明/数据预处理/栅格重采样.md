# 数据预处理 / 栅格重采样

## 功能说明

调整栅格数据的空间分辨率，将栅格重采样到指定的目标宽高。

## 输入与输出

- 输入：栅格数据（GeoTIFF）
- 输出：重采样后的栅格数据，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入栅格 | RasterData | 是 | 无 | 输入栅格数据 |
| new_width | int | 是 | 无 | 目标宽度（像素） |
| new_height | int | 是 | 无 | 目标高度（像素） |
| method | 枚举 | 否 | `Nearest` | 重采样方法：Nearest / Bilinear |

## 算法内容

重采样通过建立目标像素到源像素的映射关系，将源栅格的像元值重新分配到目标网格中。

### 像素映射

目标像素 `(x, y)` 映射到源坐标：

```
src_x = (x + 0.5) * (input_width / new_width) - 0.5
src_y = (y + 0.5) * (input_height / new_height) - 0.5
```

### Nearest（最近邻）

取源坐标最近的整数像素值。速度快，不引入新值，适合分类数据和离散值栅格。

```
px = round(src_x)
py = round(src_y)
value = input[py][px]
```

### Bilinear（双线性）

取源坐标周围 4 个像素的加权平均值。权重由源坐标到 4 个整数像素的距离决定。适合连续值栅格，结果更平滑。

```
value = v00*(1-fx)*(1-fy) + v10*fx*(1-fy) + v01*(1-fx)*fy + v11*fx*fy
```

其中 `fx`、`fy` 为源坐标的小数部分。

### GeoTransform 更新

重采样后更新 GeoTransform 的像素尺寸：

```
gt[1] = gt[1] * (input_width / new_width)   // X 方向像素尺寸
gt[5] = gt[5] * (input_height / new_height)  // Y 方向像素尺寸
```

## 处理流程

1. 验证输入参数（目标尺寸必须为正数）。
2. 计算缩放比例。
3. 更新 GeoTransform。
4. 逐波段逐像素重采样。
5. 返回重采样结果。

## 输出说明

- 输出栅格波段数与输入一致
- 输出空间参考与输入一致
- GeoTransform 的像素尺寸按缩放比例调整

## 结果解释

- Nearest 重采样保持原始像元值不变，适合分类数据
- Bilinear 重采样产生平滑过渡，适合连续值数据
- 缩小分辨率时信息会丢失，放大分辨率时不会增加细节

## 适用场景

- 统一多源数据的分辨率
- 为模型推理调整输入尺寸
- 降低数据量以加速后续处理

## 推荐输出文件名

- `xxx_resampled.tif`

## 注意事项

- 目标宽高必须为正整数
- Nearest 适合分类值和离散数据，Bilinear 适合连续值数据
- Bilinear 在边界处会自动 clamp 到有效范围
- 重采样不改变波段数和空间参考
