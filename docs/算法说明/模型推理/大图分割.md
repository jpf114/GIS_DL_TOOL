# 模型推理 / 大图分割

## 功能说明

对超出模型输入尺寸的大幅遥感影像进行滑窗推理，通过重叠区混合消除接缝，并支持将分割结果矢量化输出。

## 输入与输出

- 输入：栅格数据（GeoTIFF）
- 输出：分割掩膜栅格（`.tif`）+ 矢量多边形（`.shp` / `.geojson`）

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| tile_size | int | 否 | `512` | 切片大小（像素） |
| stride | int | 否 | `384` | 滑窗步长（像素），stride < tile_size 时产生重叠 |
| target_channels | int | 否 | `3` | 模型输入通道数 |
| normalize_mode | 枚举 | 否 | `ImageNet` | 归一化模式：None / ImageNet / MinMax01 / ZScore / Custom |
| input_is_uint8 | bool | 否 | `true` | 输入数据是否为 uint8 类型 |
| mean_r | float | 否 | `0.485` | ImageNet R 通道均值 |
| mean_g | float | 否 | `0.456` | ImageNet G 通道均值 |
| mean_b | float | 否 | `0.406` | ImageNet B 通道均值 |
| std_r | float | 否 | `0.229` | ImageNet R 通道标准差 |
| std_g | float | 否 | `0.224` | ImageNet G 通道标准差 |
| std_b | float | 否 | `0.225` | ImageNet B 通道标准差 |
| mask_threshold | float | 否 | `0.5` | 掩膜二值化阈值 |
| target_class | uint8 | 否 | `1` | 矢量化目标类别 |
| blend_mode | 枚举 | 否 | `Gaussian` | 混合模式：None / Linear / Gaussian |
| gaussian_sigma | float | 否 | `0.5` | Gaussian 混合的 sigma 系数 |
| skip_nodata | bool | 否 | `true` | 跳过全 NoData 的切片 |
| nodata_value | float | 否 | `0.0` | NoData 判定值 |
| min_polygon_area | double | 否 | `100.0` | 最小多边形面积过滤阈值 |
| simplify_tolerance | double | 否 | `1.0` | 矢量简化容差 |
| fix_topology | bool | 否 | `true` | 是否修复拓扑异常 |

## 算法内容

大图分割采用滑窗推理策略，将大幅影像按 `tile_size` 切分为多个小块，以 `stride` 为步长滑动。当 `stride < tile_size` 时，相邻切片之间存在重叠区域，通过混合权重函数消除接缝。

### 混合模式

- **None**：不混合，重叠区域直接覆盖。推理后对每个像素做 Sigmoid + Argmax 得到类别，直接写入结果。
- **Linear**：线性混合，重叠区域权重从 0 线性过渡到 1。权重公式：`w = (i + 1) / (overlap + 1)`，其中 `i` 为距边缘的像素距离。
- **Gaussian**：高斯混合，重叠区域权重按高斯函数过渡。权重公式：`w = 1 - exp(-d² / (2σ²))`，其中 `d` 为距边缘距离，`σ = gaussian_sigma × overlap`。

混合时对每个类别通道分别累加加权概率，最终按最大概率确定像素类别。

### 矢量化流程

1. 掩膜转矢量：8 方向 Moore 邻域追踪提取外环和内环
2. 过滤小多边形：按 `min_polygon_area` 移除面积过小的多边形
3. 简化：Douglas-Peucker 算法减少顶点
4. 拓扑修复：通过 GEOS Buffer(0) 修复非法几何
5. 属性添加：为每个多边形添加 class_id、feature_id、area、area_px、source 属性

## 处理流程

1. 加载模型并创建推理引擎。
2. 判断影像是否需要分块（宽或高 > tile_size）。
3. 若不需要分块，直接推理并输出。
4. 若需要分块：
   a. 计算所有切片位置和重叠信息。
   b. 预分配累加缓冲区和权重和缓冲区。
   c. 计算混合权重向量。
   d. 逐块处理：提取切片 → 预处理 → 推理 → 混合写入。
   e. 跳过全 NoData 切片（若启用）。
   f. 对累加结果做 Argmax 得到最终类别掩膜。
5. 将掩膜转为栅格输出。
6. 若需矢量化：掩膜转矢量 → 过滤 → 简化 → 拓扑修复 → 添加属性。

## 输出说明

- **栅格输出**：单波段 Byte 类型 GeoTIFF，像元值为类别编号。
- **矢量输出**：面类型 Shapefile 或 GeoJSON，每个多边形包含 class_id、feature_id、area、area_px、source 属性。

## 结果解释

- 栅格中每个像元的值表示该像素的预测类别编号（0 为背景）。
- 矢量中每个多边形代表一个连通区域，area 为地理面积，area_px 为像素面积。
- `SegmentationStats` 包含 total_tiles、skipped_tiles、inferred_tiles、total_inference_time_ms、polygon_count 等统计信息。

## 适用场景

- 大幅遥感影像的建筑物、道路、水体等地物提取
- 超出 GPU/CPU 内存限制的影像分割
- 需要矢量输出用于 GIS 分析的场景

## 推荐输出文件名

- `xxx_seg.tif`
- `xxx_seg.shp`

## 注意事项

- stride 必须 ≤ tile_size，否则自动调整为 tile_size（无重叠）
- Gaussian 混合效果最佳但计算量略高于 None
- 重叠区域越大（stride 越小），接缝越不明显，但推理次数越多
- 矢量化时 target_class 指定要提取的类别，默认为 1
- 拓扑修复使用 GEOS Buffer(0)，可能改变几何形状
