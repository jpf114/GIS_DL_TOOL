# 模型推理 / 批量推理

## 功能说明

对目录下多个 TIFF 文件执行批量推理，支持多线程并行处理，每个工作线程拥有独立的推理引擎实例。

## 输入与输出

- 输入：输入目录（包含 `.tif` / `.tiff` 文件）或文件路径列表
- 输出：输出目录，每个输入文件生成对应的分割结果

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| model_path | 文件路径 | 是 | 无 | ONNX 模型文件路径 |
| input_dir | 目录路径 | 是（目录模式） | 无 | 输入目录 |
| input_files | 文件路径列表 | 是（文件列表模式） | 无 | 输入文件路径列表 |
| output_dir | 目录路径 | 是 | 无 | 输出目录 |
| num_threads | int | 否 | `1` | 工作线程数 |
| generate_shp | bool | 否 | `true` | 是否生成矢量输出 |

## 算法内容

批量推理采用线程池模式，每个工作线程创建独立的 `RasterSeg` 实例（包含独立的 ONNX Runtime 会话），通过原子索引实现无锁任务分发。

### 线程模型

- 单线程模式（`num_threads=1`）：直接在主线程顺序处理。
- 多线程模式（`num_threads>1`）：创建 `num_threads` 个工作线程，每个线程循环获取下一个待处理文件索引，直到所有文件处理完毕。

### 文件扫描

目录模式下自动扫描 `.tif` 和 `.tiff` 后缀的文件（不区分大小写），忽略其他格式。

### 输出命名

每个输入文件 `<stem>.tif` 生成：
- `<stem>_seg.tif`：分割掩膜栅格
- `<stem>_seg.shp`：矢量多边形（若 `generate_shp=true`）

## 处理流程

1. 扫描输入目录或接收文件列表。
2. 创建输出目录（若不存在）。
3. 根据线程数选择执行模式：
   - 单线程：顺序调用 `RasterSeg::SegmentToFile()`
   - 多线程：创建工作线程池，每个线程独立推理
4. 每个文件的处理流程同单图分割。
5. 汇总所有结果，返回 `BatchResult` 列表。

## 输出说明

每个文件生成一个 `BatchResult`：

| 字段 | 说明 |
|------|------|
| input_path | 输入文件路径 |
| output_tif_path | 输出栅格路径 |
| output_shp_path | 输出矢量路径 |
| success | 是否成功 |
| inference_time_ms | 推理耗时（毫秒） |
| error_message | 错误信息（失败时） |

## 结果解释

- `success=true` 表示该文件处理成功
- `success=false` 时查看 `error_message` 了解失败原因
- `inference_time_ms` 仅包含模型推理时间，不含 IO 和后处理

## 适用场景

- 批量处理卫星影像数据集
- 大规模遥感影像解译项目
- 需要并行加速的多文件处理

## 推荐输出文件名

- `<stem>_seg.tif`
- `<stem>_seg.shp`

## 注意事项

- 线程数建议不超过 CPU 核心数，过多线程会导致资源竞争
- 每个线程会创建独立的 ONNX Runtime 会话，内存占用与线程数成正比
- 单个文件处理失败不会影响其他文件
- 进度回调通过互斥锁保护，确保线程安全
- 输出目录会自动创建，无需预先存在
