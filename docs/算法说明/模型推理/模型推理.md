# 模型推理 / 模型推理

## 功能说明

基于 ONNX Runtime 执行语义分割模型推理，将栅格数据转换为张量输入模型，经后处理输出类别掩膜。

## 输入与输出

- 输入：栅格数据（GeoTIFF）+ ONNX 模型文件
- 输出：类别掩膜栅格（`.tif`）或矢量多边形

## 参数

### 预处理参数 (PreprocessConfig)

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| target_width | int | 否 | `512` | 模型输入宽度 |
| target_height | int | 否 | `512` | 模型输入高度 |
| target_channels | int | 否 | `3` | 模型输入通道数 |
| normalize_mode | 枚举 | 否 | `ImageNet` | 归一化模式 |
| input_is_uint8 | bool | 否 | `true` | 输入是否为 uint8 |
| mean_r | float | 否 | `0.485` | R 通道均值 |
| mean_g | float | 否 | `0.456` | G 通道均值 |
| mean_b | float | 否 | `0.406` | B 通道均值 |
| std_r | float | 否 | `0.229` | R 通道标准差 |
| std_g | float | 否 | `0.224` | G 通道标准差 |
| std_b | float | 否 | `0.225` | B 通道标准差 |

### 后处理参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| mask_threshold | float | 否 | `0.5` | 掩膜二值化阈值 |
| target_class | uint8 | 否 | `1` | 矢量化目标类别 |

## 算法内容

### 预处理

将栅格数据转换为模型所需的 NCHW 张量格式。处理步骤：

1. **通道选取**：取前 `target_channels` 个波段，不足的补零。
2. **空间缩放**：将栅格 Resize 到 `target_width × target_height`，使用最近邻插值。
3. **归一化**：按 `normalize_mode` 执行归一化。

归一化模式说明：

| 模式 | 公式 | 说明 |
|------|------|------|
| None | — | 不做归一化 |
| ImageNet | `val / 255 → (val - mean) / std` | 先除 255（uint8 时），再减均值除标准差 |
| MinMax01 | `(val - min) / (max - min)` | 按波段实际最小最大值归一化到 [0,1] |
| ZScore | `(val - mean) / std` | 按波段实际均值和标准差标准化 |
| Custom | `val / 255 → (val - mean) / std` | 与 ImageNet 相同，允许自定义均值和标准差 |

4. **NaN 处理**：将 NaN 像元替换为 0。

### 推理

通过 ONNX Runtime 执行模型前向计算：

1. `ModelManager` 加载 ONNX 模型文件，创建推理会话。
2. `InferenceEngine::Run()` 将预处理后的张量送入模型。
3. 输出形状为 `[1, num_classes, H, W]` 的 logits 张量。

### 后处理

1. **Sigmoid + Argmax**：对每个像素的各类别 logit 做 Sigmoid 变换后取最大概率类别。
2. **阈值掩膜**：对单类别输出，Sigmoid 后按 `mask_threshold` 二值化。
3. **掩膜转栅格**：将 uint8 掩膜转为单波段 Byte 类型 RasterData。

## 处理流程

1. 加载模型：`ModelManager::LoadModel()` 创建 ONNX 会话。
2. 预处理：`Preprocess::RasterToTensor()` 将栅格转为 NCHW 张量。
3. 推理：`InferenceEngine::Run()` 执行模型前向计算。
4. 后处理：`Postprocess::SigmoidArgmax()` 得到类别掩膜。
5. 输出：`Postprocess::MaskToRaster()` 转为栅格数据。

## 输出说明

- 单波段 Byte 类型栅格，像元值为类别编号（0 ~ num_classes-1）。
- 输出空间参考与输入一致。

## 结果解释

- 像元值 0 通常表示背景类
- 像元值 1, 2, ... 表示各类目标
- 输出尺寸与模型输出一致（可能与输入不同）

## 适用场景

- 单幅影像的语义分割
- 模型输入尺寸内的影像推理
- 作为大图分割的基础推理单元

## 推荐输出文件名

- `xxx_seg.tif`

## 注意事项

- 模型必须为 ONNX 格式，输入为 NCHW 格式
- 归一化参数必须与模型训练时一致，否则结果不准确
- `input_is_uint8` 影响是否先除 255，需根据输入数据类型正确设置
- ONNX Runtime 在 Windows Debug 模式下可能存在兼容性问题，建议使用 Release 模式
