# 矢量处理 / 矢量拓扑检查

## 功能说明

对矢量图层进行拓扑规则检查，包括重叠、缝隙和有效性检查。基于 GEOS 库实现。

## 输入与输出

- 输入：面矢量数据
- 输出：拓扑问题列表（vector\<TopologyIssue\>）

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入矢量 | VectorData | 是 | 无 | 输入矢量数据 |
| tolerance | double | 否 | 无 | 缝隙检查容差，仅在 CheckGaps 和 FullCheck 中使用 |

### TopologyIssue 结构

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| feature_index | int | 存在问题的要素索引 |
| issue_type | string | 问题类型（overlap / gap / invalid） |
| description | string | 问题描述 |

## 算法内容

使用 GEOS 库实现矢量拓扑检查。

### 类：VectorTopology

#### CheckOverlaps(VectorData) -> vector\<TopologyIssue\>

检查矢量图层中要素之间的重叠。遍历所有要素对，使用 GEOS 判断是否存在交集区域，若交集面积大于 0 则记录为重叠问题。

#### CheckGaps(VectorData, tolerance) -> vector\<TopologyIssue\>

检查矢量图层中要素之间的缝隙。计算所有要素的并集与外接多边形的差集，若差集中存在面积大于 tolerance 的区域，则记录为缝隙问题。

#### CheckValidity(VectorData) -> vector\<TopologyIssue\>

检查矢量图层中每个要素的几何有效性。使用 GEOS 的 `GEOSisValid_r` 判断几何是否有效，无效几何记录为有效性问题。

#### FullCheck(VectorData, tolerance) -> vector\<TopologyIssue\>

依次执行重叠检查、缝隙检查和有效性检查，汇总所有拓扑问题。

### 处理细节

- 使用 GEOS 的 `GEOSIntersects_r` 和 `GEOSIntersection_r` 检测重叠。
- 使用 GEOS 的 `GEOSUnaryUnion_r` 和 `GEOSDifference_r` 检测缝隙。
- 使用 GEOS 的 `GEOSisValid_r` 检测有效性。
- 使用 GEOS 上下文管理器确保线程安全。

## 处理流程

1. 验证输入数据非空。
2. 创建 GEOS 上下文。
3. 根据调用方法执行对应检查：
   - **重叠检查**：遍历要素对，计算交集，记录重叠问题。
   - **缝隙检查**：计算并集与外接多边形的差集，记录缝隙问题。
   - **有效性检查**：逐要素验证几何有效性，记录无效问题。
   - **全量检查**：依次执行以上三项检查。
4. 返回拓扑问题列表。

## 输出说明

- 输出为 TopologyIssue 向量
- 每个 TopologyIssue 包含要素索引、问题类型和描述
- 无问题时返回空向量

## 结果解释

- issue_type 为 overlap 表示要素之间存在重叠区域
- issue_type 为 gap 表示要素之间存在缝隙
- issue_type 为 invalid 表示要素几何无效（如自相交）
- feature_index 指向存在问题的要素

## 适用场景

- 数据入库前的拓扑质量检查
- 地籍调查中地块重叠与缝隙检测
- 行政区划边界完整性验证
- 矢量数据生产环节的质量控制

## 推荐输出文件名

- `xxx_topology_check.json`

## 注意事项

- 重叠检查为逐要素对运算，大数据量时耗时较长
- 缝隙检查依赖 tolerance 参数，容差过小可能产生大量误报
- 无效几何可能导致其他检查结果不准确，建议优先修复有效性问题
- 仅支持面要素的拓扑检查，线和点要素的部分检查项不适用
