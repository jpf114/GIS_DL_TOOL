# 矢量处理 / 矢量相交

## 功能说明

计算两个矢量图层的交集，或判断两个矢量图层是否存在交集。基于 GEOS 库实现。

## 输入与输出

- 输入：两个矢量图层（A 和 B）
- 输出：交集矢量数据（Execute）或布尔判断结果（Intersects）

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| a | VectorData | 是 | 无 | 矢量图层 A |
| b | VectorData | 是 | 无 | 矢量图层 B |

## 算法内容

使用 GEOS 库实现矢量相交计算与判断。

### 类：VectorIntersect

#### Execute(const VectorData& a, const VectorData& b) -> unique_ptr\<VectorData\>

计算两个矢量图层的交集。对图层 A 中每个要素与图层 B 中每个要素执行 GEOS 相交运算，返回交集几何。

#### Intersects(const VectorData& a, const VectorData& b) -> bool

判断两个矢量图层是否存在交集。只要图层 A 中任一要素与图层 B 中任一要素有交集，即返回 true。

### 处理细节

- 使用 GEOS 的 `GEOSIntersection_r` 计算交集几何。
- 使用 GEOS 的 `GEOSIntersects_r` 判断是否相交。
- 交集为空的要素对会被跳过（不写入输出）。
- 输出要素保留图层 A 的属性字段。
- 使用 GEOS 上下文管理器确保线程安全。

## 处理流程

1. 验证输入数据 a 和 b 均非空。
2. 创建 GEOS 上下文。
3. 逐要素对处理：
   a. 将图层 A 的要素转为 GEOS 几何。
   b. 将图层 B 的要素转为 GEOS 几何。
   c. 调用 `GEOSIntersection_r` 计算交集。
   d. 跳过交集为空的几何。
   e. 将交集结果转回 Feature 并保留属性。
4. 返回交集结果。

## 输出说明

- 输出几何为交集区域
- 输出属性继承自图层 A
- 无交集的要素对不产生输出

## 结果解释

- 输出几何为两个输入图层重叠部分的区域
- 若两图层无任何重叠，输出为空图层
- Intersects 返回 true 表示至少存在一对相交要素

## 适用场景

- 提取两个区域的重叠部分
- 叠加分析中求取公共区域
- 判断地块、行政区等是否相邻或重叠

## 推荐输出文件名

- `xxx_intersect.gpkg`

## 注意事项

- 两图层的坐标系应一致，否则结果可能不正确
- 交集计算为逐要素对运算，大数据量时耗时较长
- 交集为空的要素对会被跳过而非报错
- 输出几何类型可能与输入不同（如两个面的交集可能为线或点）
