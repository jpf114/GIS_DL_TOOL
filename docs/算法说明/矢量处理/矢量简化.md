# 矢量处理 / 矢量简化

## 功能说明

按给定容差对输入矢量几何做简化，减少顶点数量，使用 Douglas-Peucker 算法。

## 输入与输出

- 输入：线或面矢量数据
- 输出：简化后的矢量数据

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入矢量 | VectorData | 是 | 无 | 输入矢量数据 |
| tolerance | double | 是 | 无 | 简化容差，单位与坐标单位一致 |

## 算法内容

使用 GEOS 库的 `GEOSSimplify_r` 函数实现 Douglas-Peucker 简化算法。

### Douglas-Peucker 算法原理

1. 给定一条折线和容差 ε。
2. 连接折线首尾形成一条直线。
3. 找到折线上离直线最远的点。
4. 若最远距离 > ε，在该点处将折线分为两段，递归处理。
5. 若最远距离 ≤ ε，移除所有中间点，仅保留首尾。

### 处理细节

- 点要素不做简化，直接保留。
- 简化后为空几何的要素会被跳过（不写入输出）。
- 简化后保留原始属性字段。
- 使用 GEOS 上下文管理器确保线程安全。

## 处理流程

1. 验证输入数据非空，容差非负。
2. 创建 GEOS 上下文。
3. 逐要素处理：
   a. 点要素直接保留。
   b. 将要素转为 GEOS 几何。
   c. 调用 `GEOSSimplify_r` 执行简化。
   d. 跳过简化后为空的几何。
   e. 将简化结果转回 Feature 并保留属性。
4. 返回简化结果。

## 输出说明

- 输出要素类型与输入一致
- 输出属性与输入一致
- 简化后为空的要素被移除

## 结果解释

- 输出几何通常比输入更少顶点、更平滑
- 容差越大，简化程度越高，顶点越少
- 容差单位与图层坐标单位一致（如米、度）

## 适用场景

- 降低大比例尺数据的顶点复杂度
- 为前端展示或网络传输生成轻量版本
- 减少矢量数据存储空间

## 推荐输出文件名

- `xxx_simplified.gpkg`

### C++ 接口

```cpp
namespace gis_ai {

class GIS_AI_API VectorSimplify {
public:
    VectorSimplify() = default;

    // Douglas-Peucker 简化，tolerance 为简化容差
    std::unique_ptr<VectorData> Execute(const VectorData& input, double tolerance);
};

} // namespace gis_ai
```

### 使用示例

```cpp
#include "gis/vector_simplify.h"
#include "io/vector_io.h"

auto vector = gis_ai::ReadVector("input.gpkg");

// 以 2.0 容差简化矢量
gis_ai::VectorSimplify simplifier;
auto result = simplifier.Execute(*vector, 2.0);

gis_ai::WriteVector(*result, "output_simplified.gpkg");
```

## 注意事项

- 容差必须 ≥ 0，为 0 时不做简化
- 简化可能改变几何的拓扑关系（如产生自相交）
- 简化后为空的要素会被跳过而非报错
- 需要拓扑保持简化时，可先简化再通过 GEOS Buffer(0) 修复
