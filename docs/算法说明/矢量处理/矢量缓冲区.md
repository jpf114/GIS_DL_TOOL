# 矢量处理 / 矢量缓冲区

## 功能说明

按给定距离为输入要素生成缓冲区，输出新的面图层。

## 输入与输出

- 输入：点、线、面矢量数据
- 输出：缓冲区结果面图层，建议 `.gpkg` 或 `.shp`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入矢量 | VectorData | 是 | 无 | 输入矢量数据 |
| distance | double | 是 | 无 | 缓冲距离，单位与坐标单位一致 |
| quad_segments | int | 否 | `8` | 象限分段数，控制缓冲区圆弧精度 |

## 算法内容

使用 GEOS 库的 `GEOSBuffer_r` 函数生成缓冲区。

### 缓冲区原理

缓冲区是距原始几何指定距离内的所有点的集合：

- **点缓冲区**：生成以点为圆心、distance 为半径的圆面
- **线缓冲区**：生成以线为中心线、两侧各扩展 distance 的带状面，端点为半圆
- **面缓冲区**：生成面向外扩展 distance 的面（正距离）或向内收缩的面（负距离）

### 象限分段数

`quad_segments` 控制四分之一圆弧的分段数：

- 值越大，圆弧越平滑，但顶点越多
- 值为 8 时，一个完整圆由 32 段线段近似
- 值为 1 时，圆弧退化为直角

### 处理细节

- 输出固定为面类型（Polygon）
- 保留原始属性字段
- 缓冲失败的要素会被跳过并记录警告日志
- 使用 `ExtractGeosFeatures` 处理缓冲结果可能产生的多部件几何

## 处理流程

1. 验证输入数据非空。
2. 创建 GEOS 上下文。
3. 逐要素处理：
   a. 将要素转为 GEOS 几何。
   b. 调用 `GEOSBuffer_r` 生成缓冲区。
   c. 提取缓冲结果中的所有面要素。
4. 返回缓冲区结果。

## 输出说明

- 输出为面类型矢量数据
- 输出属性来自输入图层
- 缓冲失败的要素不写入输出

## 结果解释

- 正距离：向外扩展
- 负距离：向内收缩（仅对面要素有意义）
- 距离单位与图层坐标单位一致

## 适用场景

- 为道路、河流、管线生成影响范围
- 为点位生成服务半径或警戒范围
- 为面要素外扩或内缩一定距离

## 推荐输出文件名

- `xxx_buffer.gpkg`

## 注意事项

- 输入必须包含至少一个要素
- 缓冲距离单位与坐标单位一致，投影坐标系下为米，地理坐标系下为度
- 负距离缓冲仅对面要素有效，对点和线可能产生空几何
- quad_segments 值越大，圆弧越平滑但数据量越大
