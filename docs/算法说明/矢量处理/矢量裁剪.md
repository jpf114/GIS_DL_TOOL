# 矢量处理 / 矢量裁剪

## 功能说明

使用裁切图层的空间范围对目标图层做几何裁剪，只保留落在裁切区域内的部分。

## 输入与输出

- 输入：
  - 目标矢量数据（被裁切）
  - 裁切矢量数据（裁切边界）
- 输出：裁切结果矢量数据，建议 `.gpkg` 或 `.shp`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| target | VectorData | 是 | 无 | 待裁切的目标矢量数据 |
| clipper | VectorData | 是 | 无 | 裁切边界矢量数据 |

## 算法内容

使用 GEOS 库的 `GEOSIntersection_r` 函数执行几何裁剪。

### 裁切几何合并

若裁切图层包含多个要素，先将所有裁切几何合并为一个 MultiPolygon 集合，再与目标要素逐一求交。

### 裁切流程

对每个目标要素：

1. 将目标要素转为 GEOS 几何。
2. 调用 `GEOSIntersection_r` 计算交集。
3. 跳过空结果。
4. 根据交集几何类型确定输出要素类型。
5. 提取交集几何中的所有要素，保留目标属性。

### 处理细节

- 输出属性仅来自目标图层，不包含裁切图层属性
- 交集结果可能改变几何维度（如面与面的交集仍为面，线与面的交集可能为点）
- 空交集的要素被跳过

## 处理流程

1. 验证目标和裁切数据非空。
2. 创建 GEOS 上下文。
3. 构建裁切几何（单要素直接使用，多要素合并为集合）。
4. 逐目标要素：
   a. 转为 GEOS 几何。
   b. 计算交集。
   c. 跳过空结果。
   d. 提取结果要素并保留属性。
5. 销毁裁切几何。
6. 返回裁切结果。

## 输出说明

- 输出要素类型由交集几何决定
- 输出属性来自目标图层
- 裁切边界本身不写入结果

## 结果解释

- 输出几何是目标要素被裁切后的部分
- 裁切边界外的部分被移除
- 返回消息中的数量是成功写入的结果要素数

## 适用场景

- 用行政区边界裁切道路、水系、地块等矢量数据
- 从全国或全省数据中提取局部区域
- 在专题分析前统一分析范围

## 推荐输出文件名

- `xxx_clipped.gpkg`

### C++ 接口

```cpp
namespace gis_ai {

class GIS_AI_API VectorClip {
public:
    VectorClip() = default;

    // 用 clipper 裁切 target，保留交集部分
    std::unique_ptr<VectorData> Execute(const VectorData& target, const VectorData& clipper);
};

} // namespace gis_ai
```

### 使用示例

```cpp
#include "gis/vector_clip.h"
#include "io/vector_io.h"

auto target = gis_ai::ReadVector("roads.gpkg");
auto clipper = gis_ai::ReadVector("boundary.gpkg");

// 用行政区边界裁切道路数据
gis_ai::VectorClip clip;
auto result = clip.Execute(*target, *clipper);

gis_ai::WriteVector(*result, "output_clipped.gpkg");
```

## 注意事项

- 目标和裁切数据必须使用相同坐标系
- 裁切图层包含多个要素时会合并为一个几何集合
- 交集为空的要素被跳过，不报错
- GEOS 上下文创建失败时会抛出异常
